Introduction Last updated: 2023-03-06

Feed The Monster teaches your child the fundamentals of reading, which can be played in offline mode as Progressive web app.

Service worker

A Service Worker is a JavaScript file that runs in the background of a web page, independent of the web page itself. It allows developers to build offline-first web applications by caching files and assets, intercepting network requests. In this project we used service worker to cache monster assets and dynamic url's for the audios.

Workbox Config

Workbox is a JavaScript library developed by Google that simplifies the process of building Progressive Web Apps (PWAs). It provides a set of tools and libraries for managing offline caching and serving of web assets, such as JavaScript files, images, and stylesheets. With Workbox, developers can easily create service workers to cache and serve resources, handle network requests, and provide an offline experience for their users. Workbox also offers various caching strategies, including network-first, cache-first, and stale-while-revalidate, to optimize the performance of web applications. In addition to caching and offline capabilities, Workbox provides tools for pre-caching and precaching assets, routing requests to specific cache strategies, and generating manifest files for web apps. Overall, Workbox simplifies the process of building PWAs and improves the user experience by providing reliable offline access to web apps and reducing network dependencies.In this project workbox is used to notify the service worker about the update when the changes are done in any code and file.

Installation

In order to utilize the contents of this repository, we will need to install the Node Package Manager locally on a development machine using the command prompt: 

 npm install npm@latest -g

Manifest

A manifest file is a file that contains important information about a software project, such as its version number, dependencies, and configuration settings. The file is typically named "manifest.json" and is located in the root directory of the project. The manifest file serves as a blueprint for the project, providing a central location for developers to specify project details and settings. It is used by various tools and platforms, including build systems, package managers, and web browsers. For example, in web development, the manifest file is used to provide metadata about a Progressive Web App (PWA), such as its name, icons, and starting URL. This information is used by web browsers to install the PWA on a user's device and provide a seamless app-like experience. Overall, the manifest file is an essential component of modern software development, allowing developers to define and manage project settings in a standardized way

Workbox Installation

You can install Workbox via npm by running the command :

  npm install workbox-cli --save-dev

Workbox provides a variety of configuration options to customize your service worker. You can create a configuration file for Workbox by running the command: 

  npx workbox wizard

The Workbox wizard will prompt you to answer a series of questions to generate your configuration file, customize as per requirement.

After you answer all the questions, the Workbox wizard will generate a configuration file for you. The file will be named workbox-config.js

At last run workbox generateSW workbox-config.js

Service Worker

To Register the Service Worker in your main JavaScript file, you'll need to register the Service Worker by calling the navigator.serviceWorker.register() method. This method takes the path to the Service Worker file as a parameter and returns a Promise. 

  
if ("serviceWorker" in navigator) {
	navigator.serviceWorker.register('/sw.js')
	  .then(registration => {
		console.log('Service Worker registered');
	  })
	  .catch(error => {
		console.error('Service Worker registration failed:', error);
	  });
  }

To install the Service Worker in our project , you'll need to listen for the install event, which fires when the Service Worker is first installed. Inside this event listener, you can cache files and assets using the Cache API 

  
self.addEventListener("install", event => {
	self.skipWaiting();
  });

After the Service Worker is installed, it will be in a "waiting" state until it is activated. You can listen for the activate event to perform any necessary cleanup, like deleting old caches. 

  
self.addEventListener("activate", function (event) {
	event.waitUntil(self.clients.claim());
  })

In your Service Worker file, you can listen for network requests using the fetch event. Inside this event listener, you can respond to the request using a cached asset or by making a new network request. 

  
self.addEventListener("fetch", event => {
	event.respondWith(
	  caches.match(event.request)
		.then(response => {
		  if (response) {
			return response;
		  }
		  return fetch(event.request);
		})
	);
  });
